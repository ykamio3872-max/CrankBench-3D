# CrankBench-3D v1.0.0 -3DCG採用エンジンシミュレータ-
## 1. 更新履歴
- **2026-07-14**: 初版作成
## 2. 概要
- 以前別プロジェクトとして作成した、TypeScript + React + Tauriベースのエンジンシミュレータを、3DCGを用いたより視覚的に理解しやすいものへとアップグレードするプロジェクトです。
- HTML5 Canvasの2Dアニメーションで描画していたエンジンモデルを、Unity Engineを使用して3DCGモデルにアップグレードし、さらにアプリのパッケージをUnityベースのものへ変更することで、アプリケーションとしての最適化を図ります。
## 3. プロジェクトの目的
- **Unityの学習**: 自身の新たな技術習得の対象としてUnity Engineを選びました。今回は既存のアプリのワイヤーフレームを流用することで、Unityの基本操作・基礎知識の習得と習熟を行うことを目的とし、注力します。
- **モダンな開発環境の理解**: Unity Engineは、現在ゲームエンジンとしてのみでなく、様々なアプリケーションやUI設計の場でもちられている汎用性の高いツールです。現在トレンドとなっている開発環境がどんなものかを知り、またそれを習得することで、就職後に実際に活用可能なスキルへと成果物を昇華させることを目指します。
## 4. 機能要件
1. エンジンモデルの3DCG描画
2. エンジンモデルを様々な角度から確認できるよう、カメラをゲーム画面で動かすライブラリを導入する。
3. 各種パラメータ（ボア径、ストローク長、コンロッド長、回転数、圧縮比など）をスライダーで調整可能にする。また、その設定値に合わせて動的にエンジンモデルの描画を更新する。
4. Rustを使用してバックエンドで実行していた熱力学的計算（単室モデル、Wiebe関数を使用）をUnity側に組み込み、XChartsライブラリを使用して、画面右側にグラフエリアとして描画する。
5. クランク角から1番シリンダーの現在の燃焼行程を検知し、現在どの工程を実行しているのか（吸気・圧縮・燃焼・排気）をリアルタイムに表示する。
## 5. 非機能要件
1. UIについては、なるべく前回のプロジェクトで作成したアプリ（CrankBench）のものを踏襲できるよう、オブジェクトの色やフォントなどを工夫する。
2. 一からエンジンの各種諸元を入力するのは難しいため、モデルケースとしてスズキ・R06Aエンジンの諸元をプリセットとして設定しておく。
